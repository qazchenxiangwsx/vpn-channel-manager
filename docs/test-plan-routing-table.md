## 部署前提

正在跑的 `/Applications/vpnmgr.app` 伺服的是打包时拷贝的 `Contents/Resources/static`(tauri.conf.json `resources`),**仓库 `app/static` 的改动不会自动出现在旧 app 里**。测试前二选一:

1. 重打包:`desktop/app/build-dmg.sh`(正式路径);
2. 快捷验证:把 `app/static/` 下 7 个 html 拷进 `/Applications/vpnmgr.app/Contents/Resources/static/`(ServeDir 每请求读盘,免重启;注意会破 Resources 签名 seal,本机 ad-hoc 自签 app 不影响启动,介意就走 1)。

web 版(FastAPI)无此问题,StaticFiles 已带 no-cache,直接生效。

## G. 诊断层(`GET /api/diag`,桌面版,`diag.rs`,2026-07-08 追加)

> 需要重建桌面 app(Rust 代码变了,拷 static 不够)。设计原则:诊断只解释「为什么掉」,登录判据仍只认 SOCKS5 探活(命门 #1)。

- [ ] 接口:`curl 127.0.0.1:$UI_PORT/api/diag` → `{channels:[{id,state,restart_count,uptime_secs,exit_code,crash_loop,tunnel_up,tunnel_iface,wg_handshake_age,ec_client_alive,ec_kick_age}]}`;全通道并发、单通道 6s 超时(卡死容器不拖全表)。
- [ ] **死循环**:复现道彤场景(wg endpoint 无响应的通道)→ 容器节点红灯 + 「死循环:已重启 N 次 · 本次存活 X」+ 问题清单 banner;手动 stop 后残留高 restart_count **不再**标死循环。
- [ ] **wg 握手**:梓川在线 → 隧道节点显示 `wg0 · 握手 X 前`;拔掉网络/对端失联 >3 分钟 → 黄灯「握手中断」;从未握手(epoch 0)→ 「从未握手(对端无响应)」。
- [ ] **EC 踢线**:江苏有线被服务端登出后 2h 内 → 隧道节点红字「服务端踢线 · X 前」+ banner「服务端主动登出…与本机其它通道无关」;>2h 不再显示。
- [ ] **EC 客户端死**:容器里 `svpnservice` 不在(kill 后善后窗口)→ 「EC 客户端进程未运行」;⚠️ 注意 EC 登录页时 svpnservice 也活着,该信号≠已登录。
- [ ] **oss 隧道接口**:openfortivpn/openconnect 通道登录前(接口未起)→ 「隧道接口未起」红字;登录后显示接口名(ppp0/tun0)。**此探针只在 wg 上实测过,oss 其余协议待验**。
- [ ] atrust / byo:无隧道级探针(tunnel_up=null),只出容器级信号——确认不显示误导性标记。
- [ ] 降级:web 版 / 旧版 app(无 /api/diag)→ 穿透图退回基础状态灯,无报错(已用 DOM stub 对旧版实测过)。
- [ ] 轮询分离:diag 20s 一拍(每通道有 exec 开销),不拖慢 8s 主刷新。
