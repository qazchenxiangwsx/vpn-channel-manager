设备重绑：把「重新登录」当常规操作

重建容器复用同一 FAKE_HWADDR / /root 卷 / hostname，多数情况会保住登录态；但 aTrust / EasyConnect 的设备指纹不止 MAC，服务端仍可能要求重新过验证码。本工具因此内建「重新登录」通路——触发重绑时在登录页过一次即可，不卡死。

IP / CIDR命中规则启用操作
IP-CIDR 规则强制带 no-resolve：按目的 IP 直接命中、不触发本地 DNS——给没有域名的内网主机（数据库 / SMB / 打印机 / 堡垒机）用。绑定/停用/删除会写 ./prov/chan{id}.yaml 并 PUT /providers/rules/{ch} 热重载，即时生效、不断现有连接，同时并入 /clash/vpn-rules.yaml 供你的 Clash 订阅同步。

删除通道

将停止并删除容器 ，其 /root 卷与登录态一并移除，且会从 mihomo 与 Clash 订阅中摘除其域名 / IP 规则（热加载、不断现有连接）。此操作不可撤销。

删除后重建会用同一 MAC / 卷 / hostname，但深信服仍可能要求重新过验证码。
