环境诊断

检测本机 Docker 环境、网络连通性、镜像可达性等依赖条件，发现问题可一键修复。

本系统用到的全部镜像 + 下载/构建/离线命令。可拉取的支持一键拉取(本机架构);自建镜像需在仓库根构建。

起容器 / 登录 / 探活 各环节常见报错的排查与解决。蓝=照指引操作即可,黄=需注意,红=较严重。运行中报错也会经友好提示给出同款建议。

Docker 引擎未就绪 cannot connect / not running

原因底层 colima 虚拟机 / Docker 没起来或连不上,通道无法运行。

解决切到「诊断」跑全量体检,按结果启动引擎。桌面 app 自带并会自启;手动可在终端 colima start。

镜像拉取失败 / ImageNotFound unexpected EOF

原因Docker Hub CDN 直连不稳(常见 EOF 中断)或限流,镜像没下下来。

解决「镜像源」加一个国内源并启用 → 回「镜像清单」一键重拉。切勿手动加 --platform amd64(会拉错架构)。

aTrust 核心服务异常 / 反复重启 arm64 机专属

原因镜像被拉成了 amd64,在 Apple Silicon 上经 Rosetta 跑会 EINTR 崩溃(点重启也无反应)。

解决删掉该通道重建,确保拉到原生 arm64(「镜像清单」按本机架构拉取);全程别带 --platform amd64。

arm64 原生→ 正常amd64 / Rosetta→ 崩溃
noVNC 登录窗白屏 / 无法连接 EC 首启尤其慢

原因容器内桌面服务(Xvfb / x11vnc / websockify)启动慢,登录窗早于服务就绪加载 → 竞态白屏。

解决等数十秒后点登录页的「重新加载登录窗」;EC 首次启动需更久。仍不行 → 在通道详情「重启」该通道。

检测连通显示「未连通」 容器在跑也可能未连通

原因容器起来 ≠ 登录成功。系统只认 SOCKS5 探活真通才算连上;多半是还没在登录界面登录成功,或会话过期。

解决进通道「登录」用 noVNC 登录成功后再「检测连通」;账密无头类确认账号 / 网关地址无误。

容器→SOCKS5→内网未通 = 未登录
Docker 网络缺失 network not found

原因通道所需的 Docker 网络(vpnnet)还没建。

解决「诊断」体检后点「创建网络」一键修复。

镜像总拉不动 / 内网域名解析失败 隐形元凶:Clash TUN

原因宿主 Clash 开了 TUN,把容器 / 虚拟机的明文 :53 DNS 劫成 fake-ip,看着网络正常,实际解析全错。

解决拉镜像 / 排查连通期间临时关掉 Clash 的 TUN 模式;持续失败时先排除 Clash 再看别处。

容器→ :53Clash fake-ip→ 解析错
容器启动后立即退出(Exited 1) EC 偶发 · 排查中

原因多种。已排除架构问题(EC / aTrust 均原生 arm64,同镜像家族 aTrust 可正常起)。EasyConnect 守护进程日志写在容器内文件、不进标准输出,致命因暂未完全定位。

排查看该通道「日志」tab;尝试在新建向导换一个 EC 版本重建。此项仍在跟进,定位后会补上确定解法。
