                方式⑦ TUN / 透明代理去哪了？
                
Web 版（纯 Docker）做不了整机 TUN：mihomo 在容器里,tun 出不了 host 网络栈。桌面版已提供宿主级 TUN 路由接管——见「全局接管」页签的方式⑦（root 助手 + 宿主 TUN 引擎,按绑定 IP 网段加路由,与 ClashX 共存）。Web 版请用方式③系统代理或④PAC。

              

            

          

        


        
        

          
          

            
Clash 接管（方式①②）：规则必须带 no-resolve

            
让你的 Clash 不抢先在本地解析，把域名原样透传给 vpn-router，由 VPN 侧 DNS 解析——客户内网域名通常只在隧道侧才解析得到。no-resolve 只属于 Clash 路径，是“上游会抢解析”的反制。

          

        

        

          
          

            
非 Clash 接管（方式③④⑤⑥）：用 socks5h，别写 no-resolve

            
系统代理 / PAC / HTTP 代理天然把域名原样交给入口，内层 mihomo 靠 respect-rules + sniffer 自己在 VPN 侧解析；命令行用 socks5h 远程解析即可。no-resolve 是 Clash 规则的修饰，不是代理地址的一部分，别带进这些方式。
