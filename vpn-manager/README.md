# VPN 通道管理器 — demo

把每个客户的 VPN(EasyConnect / aTrust)各关进一个 Docker 容器,网页上新建通道、登录、绑定域名;
绑定的域名自动经对应 VPN 通道走,其余流量走你原有 Clash,互不影响。**全 Docker,你的 Mac 除了 Docker 本身零新增。**

## 跑起来

```bash
cd vpn-manager
./start.sh
```

首次会构建后端镜像、拉取 mihomo 镜像。起来后终端会打印:
- 管理界面 `http://127.0.0.1:<随机高位端口>`
- mihomo 分流端口 `127.0.0.1:<随机高位端口>`(给你的 Clash 接)

所有端口**高位随机、只听 127.0.0.1、写进 `.env` 后重启保持不变**。

## 用法

1. **新建通道**:填客户名、类型(EasyConnect 选 7.6.3,正好是你那台的版本)、VPN 网关地址、内网探测网址。点"创建并启动容器"。
2. **登录**:点该通道的"登录",在弹出的网页窗口里完成账号 / 短信 / 扫码(和你平时登这家 VPN 一样)。
3. **检测连通**:点"检测连通",后端经该通道的 SOCKS5 访问你填的内网探测网址,通了就变 🟢。
4. **绑定域名**:输入如 `crm-a.com` → 绑定。mihomo 热加载,立即生效、不重启、不断现有连接。
5. **接 Clash**:点右上"Clash 配置",把那段 `vpn-router` 节点 + 规则粘进你现有 Clash 配置。之后访问绑定的域名就自动走对应 VPN。

## 这个 demo 已验证 / 待你验证

**已验证(本机)**:管理栈启动、端口高位随机分配、mihomo 配置热加载、后端经 Docker 内网管理容器与探活的链路。

**只有你能验证(需要真实 EC)**:
- 真实账号 / 短信登录能否在网页窗口里完成。
- **内网域名 DNS**:确认内网域名经隧道侧解析能打开(mihomo DNS 这块可能要按你环境微调)。
- **设备绑定测试(最关键)**:登录成功 → 删除容器 → 同名重建 → 看是否要求重新验证。决定"建完通道免维护"成不成立。

## 说明 / 边界

- VPN 容器的 SOCKS5(1080)只在 Docker 内网暴露;只有 noVNC 登录端口映射到 127.0.0.1(Docker 自动分配高位随机端口)。
- 凭据用 Fernet 字段级加密,主密钥在数据卷里权限锁死的文件(容器内读不到 macOS 钥匙串,这是全 Docker 的取舍)。
- 后端挂了 `docker.sock`(能控制你的 Docker),本机单人用可接受。
- `docker compose down` 停服务;VPN 容器是后端动态建的,删通道在界面点删除,或 `docker rm -f vpn-<id>`。
- Apple Silicon:EC 镜像多为 amd64,会经模拟运行(偏慢但可用);aTrust 有 arm64 原生。
- 这是 demo:UI 用原生 HTML/JS、分流用整份配置热加载。正式版再上 React、文件型 rule-provider 增量热加载、aTrustLogin 免验证码等(见落地方案文档)。
